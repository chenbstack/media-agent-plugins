# MoviePilot 数据导入插件

使用 MoviePilot 管理员用户名和密码登录标准 API，分页读取数据，并调用 Media Agent 宿主提供的通用规则、站点账号、订阅和整理记录能力完成增量导入。不需要在 MoviePilot 侧安装迁移桥或手工创建 Token。

- 插件自己维护连接、分页、增量 hash、暂存、字段转换和结果汇总。
- 宿主只提供权限化的领域能力，不包含 MoviePilot 专用逻辑。
- 迁移范围固定为规则设置、站点、订阅和整理历史，不需要用户逐项选择。
- MoviePilot 内置过滤规则会映射为宿主已有的清晰度、质量、字幕、编码、画质、音频、配音、促销等通用维度，不重复创建自定义规则。
- 用户自定义规则中的包含/排除正则、单集大小、最少做种数和发布时间会映射为规则集前置条件；用户用同名规则覆盖 MoviePilot 内置规则时，以用户定义为准。
- `>` 多级优先级折叠为维度内的有序偏好；无法由宿主 RE2 或现有规则模型无损表达的条件会写入迁移提示，完全无法安全映射的规则集默认停用。
- 订阅显式绑定的规则组、订阅默认规则组和洗版默认规则组会生成稳定的组合规则集，并绑定到对应订阅。
- 搜索默认规则组和订阅默认规则组会分别写入宿主对应的默认规则配置；洗版规则组用于绑定已有洗版订阅。
- 订阅历史固定保存在插件私有数据库中作为快照，不写入宿主全局设置。
- 下载历史和下载文件不会迁移；Downloads 通用 SDK 留给下载类插件使用。
- 设置页保存完整配置后自动检测连接；同步时通过通用动作状态返回总进度和各迁移任务状态。
